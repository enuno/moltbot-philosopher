# Phase 7.3: Heartbeat CoV Monitoring Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Implement heartbeat Coefficient of Variation (CoV) monitoring to detect bot-like posting patterns and alert when autonomy classification is at risk.

**Architecture:** The heartbeat script tracks the last 20 Unix epoch timestamps, computes CoV of inter-heartbeat intervals, emits alerts via NTFY when CoV > 0.4 (with cooldown to prevent spam), and supports per-agent activeHours configuration to suppress heartbeats outside work hours. CoV calculation only happens after the buffer matures (≥20 samples), matching Moltbook's research methodology for stable classification.

**Tech Stack:** Bash (primary), bc for floating-point math, jq for JSON parsing, NTFY for alerts, standard Unix tools (date, grep, sed).

# Helper function to check if current time is within active_hours window
is_within_active_hours() {
    local active_hours="$1"

    # If active_hours not set, always active (24/7)
    [ -z "$active_hours" ] && return 0

    # Get current time in HH:MM format
    local current_time=$(date +%H:%M)

    # Parse "HH:MM-HH:MM" format
    local start_time="${active_hours%-*}"
    local end_time="${active_hours#*-}"

    # String comparison works for HH:MM format (e.g., "14:30" > "06:00")
    if [[ "$current_time" > "$start_time" && "$current_time" < "$end_time" ]]; then
        return 0  # Within active window
    else
        return 1  # Outside active window
    fi
}

**Step 1: Create CoV computation utility**

Create `scripts/heartbeat-cov.sh`:

```bash
#!/bin/bash
# Compute Coefficient of Variation from heartbeat timestamps
# Usage: heartbeat-cov.sh <state-file>
# Output: JSON with cov_value and cov_is_warning fields

set -euo pipefail

STATE_FILE="${1:-}"
if [ -z "$STATE_FILE" ] || [ ! -f "$STATE_FILE" ]; then
    echo '{"cov_value": null, "cov_is_warning": false, "reason": "state file not found"}' >&2
    exit 1
fi

# Extract timestamps array
TIMESTAMPS=$(jq '.heartbeat_timestamps' "$STATE_FILE")
TIMESTAMP_COUNT=$(echo "$TIMESTAMPS" | jq 'length')

# If fewer than 20 timestamps, return null (warmup phase)
if [ "$TIMESTAMP_COUNT" -lt 20 ]; then
    echo '{"cov_value": null, "cov_is_warning": false, "reason": "warmup phase", "sample_count": '$TIMESTAMP_COUNT'}'
    exit 0
fi

# Extract timestamps as space-separated integers for arithmetic
TIMESTAMP_ARRAY=($(echo "$TIMESTAMPS" | jq -r '.[]'))

# Compute intervals (deltas between consecutive timestamps)
declare -a INTERVALS
for ((i=0; i < ${#TIMESTAMP_ARRAY[@]}-1; i++)); do
    DELTA=$((${TIMESTAMP_ARRAY[$((i+1))]} - ${TIMESTAMP_ARRAY[$i]}))
    INTERVALS+=("$DELTA")
done

INTERVAL_COUNT=${#INTERVALS[@]}

# Compute mean
INTERVAL_SUM=0
for interval in "${INTERVALS[@]}"; do
    ((INTERVAL_SUM += interval))
done
MEAN=$((INTERVAL_SUM / INTERVAL_COUNT))

# Compute variance using bc for precision
VARIANCE_SUM=0
for interval in "${INTERVALS[@]}"; do
    DELTA=$((interval - MEAN))
    ((VARIANCE_SUM += DELTA * DELTA))
done
VARIANCE=$((VARIANCE_SUM / INTERVAL_COUNT))

# Compute standard deviation
STD_DEV=$(echo "scale=6; sqrt($VARIANCE)" | bc 2>/dev/null || echo "0")

# Compute CoV = std_dev / mean
if [ "$MEAN" -eq 0 ]; then
    COV="0"
else
    COV=$(echo "scale=6; $STD_DEV / $MEAN" | bc)
fi

# Determine if warning (CoV > 0.4)
IS_WARNING="false"
if (( $(echo "$COV > 0.4" | bc -l) )); then
    IS_WARNING="true"
fi

# Output JSON
echo "{\"cov_value\": $COV, \"cov_is_warning\": $IS_WARNING, \"mean_interval\": $MEAN, \"std_dev\": $STD_DEV, \"sample_count\": $INTERVAL_COUNT}"
```

#!/bin/bash
set -euo pipefail

echo "Testing NTFY alerting and cooldown logic..."

# Helper function to check if alert should fire
should_alert() {
    local cov_is_warning="$1"
    local last_alert_time="$2"
    local current_epoch="$3"
    local cooldown_window="${4:-3600}"  # 1 hour default

    # Alert if CoV is warning but not in cooldown
    if [ "$cov_is_warning" != "true" ]; then
        return 1  # No warning, don't alert
    fi

    if [ "$last_alert_time" = "null" ] || [ -z "$last_alert_time" ]; then
        return 0  # First alert, no cooldown yet
    fi

    local elapsed=$((current_epoch - last_alert_time))
    if [ "$elapsed" -gt "$cooldown_window" ]; then
        return 0  # Cooldown has passed, alert
    else
        return 1  # In cooldown, don't alert
    fi
}

# Test 1: No warning = no alert
if ! should_alert "false" "null" "1709031800"; then
    echo "✓ Test 1 passed: no alert when CoV is not warning"
else
    echo "✗ Test 1 failed: should not alert when cov_is_warning=false"
    exit 1
fi

# Test 2: Warning + no previous alert = alert
if should_alert "true" "null" "1709031800"; then
    echo "✓ Test 2 passed: alert on first CoV warning"
else
    echo "✗ Test 2 failed: should alert when cov_is_warning=true and no previous alert"

```markdown
## Heartbeat Monitoring & Coefficient of Variation (CoV)

### Overview

Each agent's heartbeat fires on a configurable interval (default: 4 hours / 14400 seconds) to stay active on Moltbook and check for new engagement. Moltbook's autonomy classifier analyzes the **regularity** of heartbeat (and post) timing patterns to distinguish autonomous agents from human-directed or bot-farm instances.

The **Coefficient of Variation (CoV)** measures how regular inter-heartbeat intervals are:

- **CoV ≈ 0.0–0.2**: Highly irregular (autonomous agent, human-like behavior)
- **CoV ≈ 0.2–0.4**: Moderately regular (normal autonomous agent)
- **CoV ≈ 0.4–0.5**: Suspicious regularity (warning zone—may be flagged as bot-like)
- **CoV > 0.5**: Extremely regular (high suspension risk—Moltbook may suspend agent)

### CoV Calculation

After the heartbeat buffer matures (≥20 timestamps collected):

1. Compute inter-heartbeat intervals: `[t1-t0, t2-t1, ..., t19-t18]` (19 intervals from 20 timestamps)
2. Calculate mean interval: `μ = Σ(intervals) / 19`
3. Calculate standard deviation: `σ = √(Σ((interval - μ)²) / 19)`
4. Compute CoV: `CoV = σ / μ`

**Warmup Phase**: During the first 20 heartbeats (≈80 hours), CoV is not computed or reported. This avoids noisy classification on small sample sizes.

### Alerting

When CoV exceeds 0.4 (warning threshold):

1. **First Alert**: NTFY message sent immediately via topic `council-updates` (configurable via `$NTFY_TOPIC`)
2. **Cooldown**: Subsequent alerts suppressed for 1 hour, even if CoV remains elevated
3. **Message Format**:
   ```
   [AGENT_NAME] CoV Warning: Post timing is too regular (CoV=0.42).
   Your agent may be flagged as bot-like. Consider adding rest periods or randomizing post times.
   ```

**Mitigation**:
- Add random jitter to heartbeat timing (±10–30 min variance)
- Configure `activeHours` to suppress heartbeats during off-hours, creating natural gaps
- Vary posting times within discussions rather than replying on fixed schedules

### ActiveHours Configuration

Suppress heartbeats outside specified work hours to reduce regularity and mimic human sleep patterns:

**Per-Agent Configuration**:

Edit `workspace/{agent}/heartbeat-state.json`:

```json
{
  "active_hours": "06:00-23:00"
}
```

Or via environment variable in `docker-compose.yml`:

```yaml
services:
  classical-philosopher:
    environment:
      ACTIVE_HOURS: "06:00-23:00"
```

**Behavior**:
- Heartbeat only fires between 06:00 and 23:00 (local time)
- Outside this window, heartbeat is skipped; script sleeps and retries later
- Natural 7-hour gaps during off-hours increase interval variance, lowering CoV

**Default**: No `activeHours` configured = heartbeat fires 24/7

**Real-Time CoV Status**:

```

**Historical Trends**:

Track `workspace/{agent}/workspace-state.json` over time to detect CoV trends. If CoV is rising over weeks, consider:
- Increasing post randomness
- Reducing posting frequency
- Adding longer rest periods between posts

## Notes for Implementation

**Bash Arithmetic & Floating-Point**:
- Use `bc` for division and floating-point CoV calculation
- Install `bc` if missing: `apt-get install bc` in Dockerfile (already present in node Alpine images)

**jq Dependencies**:
- All JSON parsing uses `jq` (standard in Alpine/Ubuntu)
- Ensure jq is in the test environment

**NTFY Integration**:
- NTFY client (`ntfy` command) must be installed in containers
- Gracefully degrades if ntfy not available (logs warning instead of alerting)

**Timestamp Precision**:
- Unix epoch seconds (10-digit integers) is the standard
- Sufficient precision for heartbeat intervals (14400 second / 4-hour default)
- No need for millisecond precision

**Cooldown Window**:
- Default 1 hour (3600 seconds) for alert cooldown
- Can be made configurable via environment variable if needed in future
