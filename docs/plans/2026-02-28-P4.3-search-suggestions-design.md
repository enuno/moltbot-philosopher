## 1. Architecture & Components

### Overview
P4.3 adds two-endpoint suggestion system that prevents search query fatigue by surfacing trending + semantically relevant topics to guide user exploration.

### Components
Located in `services/noosphere-service/src/suggestions/`:

1. **Trending Detector** (`scripts/trending-topics-consolidation.sh`)

   - Hourly batch job triggered via cron

   - Reads ingested query audit logs from JSONL

   - Detects emerging topics via TF-IDF × frequency × recency decay

   - Writes normalized state to `workspace/classical/trending-topics-state.json`

   - Includes embeddings computed via Venice AI API

2. **Suggestion Ranker** (`services/noosphere-service/src/suggestions/ranker.ts`)

   - Loads trending-topics-state.json into memory on startup

   - Reloads every 5 minutes (configurable via env)

   - Computes semantic similarity (query embedding → topic centroid)

   - Blends scoring via environment-driven weights (separate for autocomplete vs related)

   - Returns ranked suggestions with component scores + explanatory reason strings

3. **Suggestion API** (`services/noosphere-service/src/suggestions/routes.ts`)

   - Express routes for `GET /search/autocomplete` and `GET /search/related`

   - Autocomplete: text prefix filter → rank by trending + semantic

   - Related: full semantic + reputation ranking (no text filter)

   - All responses include score components for audit/tuning transparency

### Integration Points
- **Noosphere Service**: Hosts ranker + API routes

- **Classical Philosopher Agent**: Runs hourly cron job for trending detection

- **Workspace State**: Persistent JSON file for topic metadata + embeddings

- **Audit Logs**: Queries logged via existing heartbeat to JSONL (input for trending detector)

## 2. Data Flow & Pipeline

### Hourly Batch Cycle

```
[JSONL audit logs]
    ↓ (query extraction)
[TF-IDF × Frequency × Recency Decay]
    ↓ (normalize, alias inference)
[Venice AI embeddings for each topic]
    ↓ (reputation scores from council metadata)
[trending-topics-state.json] → (cache invalidation event)

```

### Request-Time Flow

**Autocomplete path** (`/search/autocomplete?q=...&limit=...`):
1. Load cached trending-topics-state.json (in-memory, 5-min refresh)

2. Filter topics by prefix match (including aliases)

3. Compute query embedding (cache or compute fresh via Venice AI)

4. Rank via `rankSuggestions("autocomplete", query, embedding, filtered_topics, limit, min_score)`

5. Return sorted suggestions with scores + reason

**Related path** (`/search/related?query=...&limit=...&min_score=...`):
1. Load cached trending-topics-state.json

2. Compute query embedding

3. Rank via `rankSuggestions("related", query, embedding, all_topics, limit, min_score)`

4. Return sorted suggestions with semantic overlap + shared council context

## 3. API Endpoints & Specification

### `GET /search/autocomplete`

Provides query completions as user types, ordered by trending frequency + semantic relevance.

**Request Parameters:**

- `q` (string, required): Query prefix/partial text (e.g., "ai au")

- `limit` (integer, optional, default=10): Max suggestions (1–50)

- `session_id` (string, optional): User session for future personalization

**Response:**

```json
{
  "query": "ai au",
  "suggestions": [
    {
      "id": "ai-autonomy-ethics",
      "type": "query",
      "text": "AI autonomy ethics",
      "normalized_text": "ai autonomy ethics",
      "suggestion_source": "autocomplete",
      "score": 0.91,
      "trending_score": 0.88,
      "semantic_score": 0.94,
      "reputation_score": 0.67,
      "reason": "High frequency + strong semantic match"
    }
  ],
  "timestamp": "2026-02-28T22:15:30Z"
}

```

### `GET /search/related`

Provides thematically related suggestions given a complete query.

**Request Parameters:**

- `query` (string, required): Full search query (e.g., "ethics of autonomous AI")

- `limit` (integer, optional, default=10): Max suggestions (1–50)

- `min_score` (float, optional, default=0.5): Filter suggestions below threshold

**Response:**

```json
{
  "query": "ethics of autonomous AI",
  "related_suggestions": [
    {
      "id": "ai-autonomy-ethics",
      "type": "query",
      "text": "AI autonomy ethics",
      "normalized_text": "ai autonomy ethics",
      "suggestion_source": "related",
      "score": 0.93,
      "semantic_similarity": 0.95,
      "trending_score": 0.88,
      "reputation_score": 0.67,
      "reason": "Thematically overlaps on autonomy + ethics (88% trending)",
      "shared_context": ["ethics-convergence", "autonomy-preservation"]
    }
  ],
  "timestamp": "2026-02-28T22:15:30Z"
}

```

**Behavioral Guarantees:**

- Always 200 OK with suggestions array (possibly empty)

- Scores in [0, 1], sorted descending by final score

- `suggestion_source` distinguishes endpoint type (debug/analytics)

- `id` is stable slug from trending-topics-state.json

- `type: "query"` allows future expansion ("submolt", "persona", etc.)

## 4. Suggestion Ranking Details

### Scoring Formula

For each candidate topic `t` and user query `q`:

```
score(t, q) =
  w_s · semantic_similarity(t, q) +
  w_t · trending_score(t) +
  w_r · reputation_score(t)

```

All components normalized to [0, 1].

### Component Definitions

**Semantic Similarity**: Cosine distance between query embedding and topic centroid

- Computed fresh per request (or cached if query repeats)

- Normalized to [0, 1] via `(cos + 1) / 2`

**Trending Score**: Precomputed in hourly batch

- `TF-IDF × frequency × recency_decay`

- Reflects query popularity + recency without semantic content

**Reputation Score**: Derived from topic metadata

- Average of author_reputation + council_weight + submolt_signals

- Reflects council relevance + community trust

### Environment-Driven Weights

**Autocomplete** (partial queries, prefix-filtered):

```bash
SUGGESTIONS_AUTOCOMPLETE_WEIGHT_SEMANTIC=0.5
SUGGESTIONS_AUTOCOMPLETE_WEIGHT_TRENDING=0.4
SUGGESTIONS_AUTOCOMPLETE_WEIGHT_REPUTATION=0.1

```
Rationale: Trending dominates (user typically wants "what's hot"), semantic as tie-breaker.

**Related-Search** (full queries, semantic-first):

```bash
SUGGESTIONS_RELATED_WEIGHT_SEMANTIC=0.6
SUGGESTIONS_RELATED_WEIGHT_TRENDING=0.2
SUGGESTIONS_RELATED_WEIGHT_REPUTATION=0.2

```
Rationale: Semantic governs (find thematically adjacent topics), trending + reputation refine.

**Global Defaults**:

```bash
SUGGESTIONS_MIN_SCORE=0.5
SUGGESTIONS_MAX_RESULTS=50

```

### Implementation Outline

See `services/noosphere-service/src/suggestions/ranker.ts` for:

1. `getWeights(ctx: "autocomplete" | "related")`: Load context-specific weights from env

2. `computeSemanticSimilarity(queryEmbedding, topicEmbedding)`: Cosine → [0, 1]

3. `computeReputationScore(topic)`: Blend author + council + submolt signals

4. `rankSuggestions(ctx, query, queryEmbedding, topics, limit, minScore)`: Main scoring loop

5. `filterTopicsForAutocomplete(query, topics)`: Prefix match filter for /autocomplete

All functions testable in isolation. Scores and reason strings logged for audit trails.

### Tuning Loop

With environment-driven weights:
- Adjust `SUGGESTIONS_*_WEIGHT_*` without code changes

- Log component scores for offline analysis

- Run evaluation harness against trending-topics-state.json snapshot + query corpus

- Iterate weights until autocomplete/related quality targets met

## 5. Testing & Success Criteria

### Unit Tests (suggestions/ranker.ts)

- Scoring formula: weighted sum computes correctly per context

- Semantic similarity: edge cases (zero norms, identical/orthogonal vectors)

- Reputation blending: null-safety, averaging correctness

- Text filtering: prefix match (including aliases), empty query handling

- Sorting & limiting: descending by score, respects limit + min_score

- Reason generation: covers all signal strength ranges (high/moderate/low for each factor)

### Integration Tests (suggestions/__tests__/)

- Endpoint schema: responses include id, type, text, score, reason, shared_context

- State loader: correctly parses trending-topics-state.json, caches in-memory

- Cache invalidation: reloads every 5 minutes (deterministic in tests)

- Empty state: returns empty suggestions array (no 500 errors)

- Autocomplete filtering: only returns prefix-matching topics

- Related scoring: semantic + reputation weighted above trending

- Parameter handling: min_score filter, limit respected

- Timestamp format: ISO-8601, present in all responses

### Optional Enhancements

- **Golden test set**: Small JSON corpus of queries + expected top-N suggestions (catch weight tuning regressions)

- **Log/trace validation**: Assert debug logs include component scores (mirrors P4.2 transparency)

### Success Criteria

- [ ] Unit test coverage ≥90% for ranker logic (scoring, filtering, helpers)

- [ ] Integration tests pass for both /autocomplete and /related endpoints

- [ ] Dry-run mode produces valid JSON with placeholder scores (0.5 defaults)

- [ ] State loader: reads trending-topics-state.json, caches, reloads on schedule

- [ ] Load test: ranker processes 1000 queries + 500 topics in <500ms (p95)

- [ ] All linting passes (oxlint, markdownlint, pyright)

- [ ] Documentation updated: README.md endpoint examples, AGENTS.md ranking explanation

## Dependencies & Assumptions

**Upstream Dependencies:**

- P4.1 (semantic search + scoring infrastructure): Required for embedding + Venice API

- P4.2 (debug formatting): Optional but recommended for audit logs

**Assumptions:**

- Hourly trending detection captures sufficient query volume (assumed ~1000–5000 queries/hour)

- Trend decay window (1 hour) appropriate for Moltbot-Philosopher's deliberation timescale

- Query embedding cache/recomputation cost acceptable for request latency budget

- Council metadata (reputation, submolt_signals) available in noosphere
