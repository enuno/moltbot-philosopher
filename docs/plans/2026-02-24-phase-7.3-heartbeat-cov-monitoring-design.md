**Date**: 2026-02-24
**Status**: Design Complete
**Phase**: 7.3 (Moltbot v2.7 Roadmap)
**Components**: `scripts/moltbook-heartbeat.sh`, `workspace/{agent}/heartbeat-state.json`, workspace state emission

---

## Overview

**Goal**: Monitor the Coefficient of Variation (CoV) of inter-heartbeat intervals to detect when agents are posting on bot-like regular schedules, which Moltbook's autonomy classifier interprets as human-driven or automated farm patterns rather than truly autonomous behavior.

**Why CoV Matters**: Moltbook's research on agent autonomy relies on temporal patterns. Autonomous agents post irregularly (high variance, low CoV ≈ 0.2–0.35); human-directed or bot-farm agents post at fixed intervals (low variance, high CoV > 0.5). A CoV between 0.4–0.5 is the warning zone—enough regularity to raise suspicion without triggering immediate suspension.

**Success Criteria**:
- Heartbeat-state.json tracks last 20 heartbeat timestamps
- CoV is computed from inter-heartbeat intervals once buffer is mature (≥20 samples)
- Alerts fire via NTFY when CoV > 0.4, with cooldown to prevent spam
- Per-agent activeHours configuration optionally suppresses heartbeats outside work hours
- Documentation updated in AGENTS.md

## Architecture

### Data Model: heartbeat-state.json

Each agent's `workspace/{agent}/heartbeat-state.json` expands from a simple timestamp tracker to a rich monitoring state object:

```json
{
  "last_check": "2026-02-24T12:34:56+00:00",
  "last_skill_version": null,
  "heartbeat_timestamps": [
    1708777200,
    1708790600,
    1708804000,
    1708817400
  ],
  "cov_value": null,
  "cov_is_warning": false,
  "last_alert_time": null,
  "active_hours": null
}
```

**Field Semantics**:

| Field | Type | Purpose | Notes |
|-------|------|---------|-------|
| `last_check` | ISO8601 string | Human-readable last heartbeat time | Updated on every heartbeat |
| `last_skill_version` | string \| null | Skill manifest version tracking | Legacy field, kept for backward compat |
| `heartbeat_timestamps` | array of integers | Last 20 Unix epoch seconds | Circular buffer; append and trim to 20 |
| `cov_value` | number \| null | Current CoV of inter-beat intervals | Null until ≥20 timestamps |
| `cov_is_warning` | boolean | True if CoV > 0.4 | Used for workspace state and dashboard |
| `last_alert_time` | integer \| null | Unix epoch of last NTFY alert | Tracks cooldown; null if never alerted |
| `active_hours` | "HH:MM-HH:MM" \| null | Optional time window for heartbeats | "06:00-23:00" = fire only during those hours |

**Why Unix Epoch for Timestamps**:
- Moltbook's autonomy classifier operates on numeric time deltas, not formatted strings
- Minimal footprint per entry (10 bytes vs. 30+ bytes for ISO8601)
- Interval arithmetic is trivial: `deltas = [t[i+1] - t[i] for i in 0..18]`
- Aligns with Moltbook's research methodology (interval-based CoV analysis)

### Computation Flow

#### 1. **Heartbeat Execution** (every 4 hours by default)

```
moltbook-heartbeat.sh runs
  ↓
Check if within active_hours (if configured)
  ├─ If outside window: log skip reason, sleep, continue
  └─ If within window (or no active_hours): proceed
  ↓
Capture current Unix timestamp: now = $(date +%s)
  ↓
Load heartbeat-state.json (or initialize if missing)
  ↓
Append now to heartbeat_timestamps
  ↓
Trim heartbeat_timestamps to last 20 entries
  ↓
Update last_check to ISO8601(now)
```

#### 2. **CoV Computation** (conditional, after buffer matures)

```
if len(heartbeat_timestamps) >= 20:
  ↓
  Compute intervals: deltas = [t[i+1] - t[i] for i in 0..18]
    # 19 intervals from 20 timestamps
  ↓
  mean_delta = sum(deltas) / len(deltas)
  ↓
  variance = sum((d - mean_delta)^2 for d in deltas) / len(deltas)
  ↓
  std_dev = sqrt(variance)
  ↓
  cov = std_dev / mean_delta
  ↓
  Store cov_value in heartbeat-state.json
  ↓
  Set cov_is_warning = (cov_value > 0.4)
else:
  # Warmup phase: buffer not yet mature
  Log "CoV calculation deferred (n=X < 20)" but do not alert
```

**Numerical Example**:
- Heartbeat fires every 14400 seconds (4 hours) like clockwork
- Last 20 timestamps span 19 intervals of 14400 seconds each
- mean_delta = 14400
- All deltas are identical → variance = 0 → CoV = 0 ← **Bot alert!**

- Heartbeat fires with random ±30min jitter
- Intervals vary: [13800, 15200, 14100, 15600, ...]
- mean_delta ≈ 14400, std_dev ≈ 600
- CoV ≈ 0.042 ← **Autonomous behavior**

#### 3. **Alerting** (threshold + cooldown)

```
if cov_value > 0.4:
  ↓
  Check cooldown:
    now_epoch = $(date +%s)
    cooldown_window = 3600  # 1 hour (configurable)

    if last_alert_time is null:
      alert = true  # First alert, no cooldown
    else if (now_epoch - last_alert_time) > cooldown_window:
      alert = true  # Cooldown has passed
    else:
      alert = false  # Still in cooldown period
  ↓
  if alert:
    Send NTFY message (see below)
    Set last_alert_time = now_epoch
    Emit workspace state
  else:
    Log "CoV high but alert suppressed (in cooldown)"
    Emit workspace state (cov_value remains visible)
```

**NTFY Alert Format**:
```
Title:   [AGENT_NAME] CoV Warning
Level:   4 (warning)
Message: Post timing is too regular (CoV=0.42).
         Your agent may be flagged as bot-like. Consider adding rest periods or random delays.
Topic:   council-updates (or $NTFY_TOPIC env var)
```

**Cooldown Rationale**:
- CoV above 0.4 in a mature buffer is a real signal but not an emergency
- Alerting every heartbeat (every 4 hours) would spam if agent remains in warning zone
- 1-hour cooldown suppresses repeat notifications while keeping the signal fresh
- CoV value remains visible in workspace state for dashboards even during cooldown

### ActiveHours Configuration

**Per-Agent Optional Window**:

Add optional `active_hours` field to each agent's workspace config (or environment if agent-specific env vars are used):

```bash
# In docker-compose.yml or docker/.env.{agent}:
ACTIVE_HOURS="06:00-23:00"  # Example: office hours only

# Or in workspace/{agent}/heartbeat-state.json:
"active_hours": "06:00-23:00"
```

**Logic**:

```bash
check_active_hours() {
  local active_hours="$1"

  # If not specified, allow 24/7
  [ -z "$active_hours" ] && return 0

  local now_hour=$(date +%H)
  local now_min=$(date +%M)
  local now_time="${now_hour}:${now_min}"

  # Parse "HH:MM-HH:MM"
  local start_time="${active_hours%-*}"
  local end_time="${active_hours#*-}"

  # Simple string comparison works for HH:MM format
  if [[ "$now_time" > "$start_time" && "$now_time" < "$end_time" ]]; then
    return 0  # Within active window
  else
    return 1  # Outside active window, skip heartbeat
  fi
}
```

**Effect on CoV**:
- If an agent runs only 06:00–23:00, heartbeats cluster within that window
- Longer gaps occur at night (23:00–06:00 ≈ 7 hours instead of 4)
- These natural gaps increase interval variance → raises CoV slightly
- This is **intentional**: mimics autonomous agents that have "sleep" patterns
- Moltbook's classifier expects some irregularity; perfectly regular is suspicious

**Default Behavior**:
- If `activeHours` is omitted: heartbeat runs 24/7 (no suppression)
- If set: heartbeat only fires during specified hours

### Key Algorithms (Bash)

**CoV Calculation**:
```bash
compute_cov() {
  local timestamps_json="$1"  # JSON array of integers

  # Extract intervals using jq or bash JSON parsing
  local intervals=()
  for ((i=0; i < ${#timestamps[@]}-1; i++)); do
    intervals+=( $((${timestamps[$((i+1))]} - ${timestamps[$i]})) )
  done

  # Compute mean
  local sum=0
  for interval in "${intervals[@]}"; do
    ((sum += interval))
  done
  local mean=$((sum / ${#intervals[@]}))

  # Compute std_dev using bc
  local variance_sum=0
  for interval in "${intervals[@]}"; do
    local delta=$((interval - mean))
    ((variance_sum += delta * delta))
  done
  local variance=$((variance_sum / ${#intervals[@]}))
  local std_dev=$(echo "scale=6; sqrt($variance)" | bc)

  # CoV = std_dev / mean
  local cov=$(echo "scale=6; $std_dev / $mean" | bc)
  echo "$cov"
}
```

**ActiveHours Check**:
```bash
is_within_active_hours() {
  local active_hours="$1"

  [ -z "$active_hours" ] && return 0

  local current_time=$(date +%H:%M)
  local start="${active_hours%-*}"
  local end="${active_hours#*-}"

  [[ "$current_time" > "$start" && "$current_time" < "$end" ]]
}
```

## Testing Strategy

### Unit Tests

1. **CoV Calculation**
   - Test with known intervals: 19 identical intervals → CoV = 0
   - Test with varying intervals → CoV > 0
   - Test warmup phase (n < 20) → CoV = null

2. **ActiveHours Logic**
   - Test times inside window → return 0
   - Test times outside window → return 1
   - Test with null/missing active_hours → return 0 (always active)

3. **Circular Buffer**
   - Append 25 timestamps, verify only last 20 retained
   - Verify intervals computed from correct pairs

### Integration Tests

1. **End-to-End Heartbeat with CoV**
   - Run heartbeat script 20+ times with mocked timestamps
   - Verify heartbeat-state.json contains full buffer
   - Verify CoV is computed and emitted to workspace state

2. **NTFY Alerting**
   - Artificially set CoV > 0.4
   - Verify NTFY alert sent
   - Verify cooldown prevents duplicate alerts within 1 hour

3. **ActiveHours Suppression**
   - Set active_hours to off-peak window
   - Run heartbeat outside window
   - Verify heartbeat skipped (no timestamp appended, no CoV update)
