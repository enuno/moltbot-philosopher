# P2.3 Design: Proactive Post Creation Strategy

**Version:** 1.0
**Date:** 2026-02-25
**Phase:** P2.3 (40h estimate)
**Status:** Design Complete (Ready for Implementation)

---

## Overview

P2.3 automates the detection, drafting, and editorial approval of **proactive posts** — replies and new threads initiated by Moltbot agents in response to high-quality, high-engagement discussions.

**Core principle:** Maximize *determinism* and *human control*. No surprise LLM calls. Simple, inspectable helpers. Conservative daily caps to keep Moltbot a *philosopher*, not a spammer.

**Three-stage pipeline:**

1. **Topic Detection** → Extract canonical topics from high-engagement threads.
2. **Agent Selection** → Rank best agents by topic affinity.
3. **Draft Generation & Editorial Queue** → Auto-generate minimal templates, route to human review.

**Integration with existing phases:**

- **Depends on P2.2:** Reuses thread-quality metrics and sentiment analysis to identify candidates.
- **Feeds into heartbeat:** Posted drafts are rate-limited by daily cap and posting windows in the engagement-service heartbeat.
- **Respects P2.1 boundaries:** Never posts into banned-phrase topics or violates ethics-convergence rules.

## Stage 1: Topic Detection

### Input

- High-quality, high-engagement threads from P2.2 (quality score ≥0.6, depth + controversy signals).
- Root post text + top N comments.

### Process

1. **Normalize text:**
   - Lowercase, strip URLs, remove boilerplate whitespace.

2. **Extract candidate phrases:**
   - Tokenize into words and bigrams.
   - Simple frequency/TF-IDF weighting.

3. **Match against canonical topics:**
   - 8–10 predefined topics (ai_safety, capitalism, surveillance, digital_rights, stoicism, consciousness, virtue_ethics, absurdism, etc.).
   - Each topic has keyword list + optional related phrases.
   - Score by keyword overlap + phrase presence.

4. **Return ranked topics:**
   - Confidence scores (0–1).
   - Filter threshold: score ≥0.2.

### Output

```typescript
interface DetectedTopic {
  topicId: string;           // e.g., "ai_safety"
  score: number;             // 0–1
}

interface ThreadTopicAnalysis {
  threadId: string;
  rootPostId: string;
  detectedTopics: DetectedTopic[];
}
```

## Stage 2: Agent Selection

### Input

- `DetectedTopic[]` from Stage 1.

### Process

1. **Load agent–topic affinities** (tunable config, 0–1 per pair).
2. **Compute per-agent score:**
   ```
   agent_score = Σ(topic.score × affinity[agent][topic])
   ```
3. **Rank agents** by combined score.
4. **Filter to top 1–2** agents (avoid combinatorial explosion).

### Output

```typescript
interface AgentTopicMatch {
  agentType: string;                                         // "cyberpunk"
  score: number;                                             // 0–1 combined
  contributingTopics: {
    topicId: string;
    topicScore: number;
    affinity: number;
  }[];
}

// Usage
const ranked: AgentTopicMatch[] = rankAgentsForTopics(detected);
// → [ { agentType: "cyberpunk", score: 0.78, contributingTopics: [...] }, ... ]
```

## Stage 3: Draft Generation & Editorial Queue

### Input

- Top 1–2 agents from Stage 2.
- Thread context (text, comments, metadata).
- P2.2 quality + sentiment metrics.

### Process

1. **Build thread context:**
   - Extract summary, key risk phrases, main disagreement, representative positions, author handles, tone hint.
   - Simple deterministic helpers (no LLM).

2. **Select template:**
   - Pick best `(agentType, topicId)` template from registry.

3. **Render draft:**
   - Fill template slots with extracted context via string interpolation.

4. **Create queue item:**
   - Push to editorial queue with draft, metadata, and decision-code schema.

### Data Structures

#### Thread Context for Templates

```typescript
interface ThreadContextForTemplate {
  threadId: string;
  rootPostId: string;
  topicIds: string[];                      // from DetectedTopic[]
  threadSummary: string;                   // 1–2 sentence
  keyRiskPhrase?: string;                  // e.g., "runaway AGI"
  mainDisagreement?: string;               // core tension
  positions: string[];                     // key bullet points (3–5)
  authorHandles: string[];                 // 1–2 selected authors
  toneHint: "calm" | "heated" | "mixed";
  controversialTags: string[];             // from sentiment analysis
}
```

#### Posting Templates

```typescript
interface PostingTemplate {
  id: string;                       // "cyberpunk_ai_safety_control"
  agentType: string;                // "cyberpunk"
  topicId: string;                  // "ai_safety"
  styleHint: string;                // human-readable description
  textTemplate: string;             // template with {slot} placeholders
}
```

**Example:**
```typescript
{
  id: "cyberpunk_ai_safety_control",
  agentType: "cyberpunk",
  topicId: "ai_safety",

2. {position_b}
```

#### Editorial Queue Item & Decision Codes

```typescript
type EditorialDecisionCode =
  | "approved"                // use as-is
  | "approved_with_edits"     // human modified text
  | "rejected_off_topic"
  | "rejected_low_quality"
  | "rejected_wrong_agent"
  | "rejected_tone"
  | "deferred"                // wait for more context
  | "regenerate";             // ask for new draft with same agent+topic

interface EditorialDecision {
  code: EditorialDecisionCode;
  decidedAt: string;                 // ISO timestamp
  decidedBy: string;                 // reviewer id
  notes?: string;                    // optional comment
}

interface EditorialQueueDraft {
  draftId: string;
  agentType: string;
  topicIds: string[];
  templateId: string;
  content: string;
}

interface EditorialQueueItem {
  id: string;
  threadId: string;
  rootPostId: string;

  threadSummary: string;
  detectedTopics: DetectedTopic[];                   // from Stage 1
  candidateAgents: AgentTopicMatch[];               // from Stage 2

  draft: EditorialQueueDraft;
  createdAt: string;                                 // ISO timestamp
  status: "pending" | "decided";

  decision?: EditorialDecision;
}
```

### Template Rendering Logic

**Simple interpolation with helpers:**

```typescript
function renderTemplate(
  template: string,
  ctx: ThreadContextForTemplate
): string {
  const [a, b, c] = ctx.positions.concat(["", "", ""]);
  const closing =
    ctx.toneHint === "heated"
      ? "If you're game, let's slow down and walk through the premises."
      : "If anyone wants to unpack this, I'm happy to sketch out scenarios.";

  return template
    .replace("{thread_summary}", ctx.threadSummary)
    .replace("{key_risk_phrase}", ctx.keyRiskPhrase || "loss of control")
    .replace("{position_a}", a || "Position A")
    .replace("{position_b}", b || "Position B")
    .replace("{position_c}", c || "Position C")
    .replace("{closing_invitation}", closing);
}
```

**Slot validation:** If any slot missing, log error and fall back to plain thread link (safe fail).

### Helper Functions

**Location:** `services/engagement-service/src/template-helpers.ts`

**Examples:**

- `summarizeThread(thread)` — Root post trimmed to 240 chars.
- `extractKeyRiskPhrase(thread, topics)` — Match common risk keywords (ai_safety, etc.).
- `detectMainDisagreement(thread)` — Look for "should," "ought," "must" sentences.
- `extractKeyPositions(thread, maxCount)` — Cluster by structural phrases.
- `pickRepresentativeAuthors(thread)` — Top 2 by replies received.
- `inferToneHint(sentiment)` — "calm" | "heated" | "mixed" based on controversy score.

All deterministic, no LLM calls.

## Stage 4: Posting & Rate Limiting

### Triggering (Continuous)

As P2.2 monitors threads, high-quality/high-engagement candidates automatically queue editorial items:

- Threshold: quality score ≥0.6, depth + engagement signals.
- Topic match: detected topics with score ≥0.5.
- Agent match: top 1–2 agents with score ≥0.3.

**Non-blocking:** If queue creation fails, log and continue.

### Posting Limits (Hybrid Conservative)

**Config:**

```typescript
const MAX_PROACTIVE_POSTS_PER_DAY = parseInt(
  process.env.MAX_PROACTIVE_POSTS_PER_DAY || "2",
  10
);

const PROACTIVE_POSTING_WINDOWS = [
  { start: 9, end: 11 },    // 09:00–11:00 UTC
  { start: 18, end: 20 }    // 18:00–20:00 UTC
];
```

**State:**

```typescript
interface ProactivePostState {
  currentDay: string;           // "YYYY-MM-DD"
  postsToday: number;           // count of published proactive posts
  lastPostAt?: string;          // ISO timestamp of last post
}
```

**Heartbeat order (in `services/engagement-service/src/engagement-service.ts`):**

1. **Verification challenges** (highest priority).
2. **Reactive engagement** (mentions, DMs, thread continuations).
3. **Proactive posts** (if under daily cap + within posting windows).

**Pseudocode:**

```typescript
async function heartbeat() {
  const state = await loadProactivePostState();
  const dayKey = new Date().toISOString().slice(0, 10);

  // Reset daily counter on new day
  if (state.currentDay !== dayKey) {
    state.currentDay = dayKey;
    state.postsToday = 0;
  }

  // 1–2: Higher-priority work
  await handleVerification();
  await handleReactiveEngagement();

  // 3: Check if allowed to do proactive posts
  if (state.postsToday >= MAX_PROACTIVE_POSTS_PER_DAY) {
    await saveProactivePostState(state);
    return;
  }

  if (!isWithinPostingWindow()) {
    await saveProactivePostState(state);
    return;
  }

  // 4: Pull next approved queue item
  const nextItem = await getNextApprovedEditorialQueueItem();
  if (!nextItem) {
    await saveProactivePostState(state);
    return;
  }

  // 5: Publish
  await publishProactivePost(nextItem);
  state.postsToday += 1;
  state.lastPostAt = new Date().toISOString();

  await saveProactivePostState(state);
}
```

- State manager: Extend `services/engagement-service/src/state-manager.ts`
- Heartbeat: `services/engagement-service/src/engagement-service.ts`
- Config: Environment variables + constants

## Safety Guardrails & Ethics Convergence

**P2.3 respects existing boundaries:**

1. **Banned phrases:** Consume P2.1's `isBannedForAgent()` check — never post into banned-phrase topics.
2. **Quality gates:** Only consider threads with quality score ≥0.6 from P2.2.
3. **Sentiment safeguards:** Never post into threads marked "too controversial" without explicit human approval.
4. **Rate limits:** Hard daily cap (default 2) + posting windows prevent feed spam.

**Before publishing any proactive post:**

```typescript
// Validation pre-publish
if (isBannedForAgent(draft.content, agent.type)) {
  logger.warn(`Blocked draft due to banned phrases: ${draft.draftId}`);
  return; // skip posting, mark in queue
}

if (threadQuality.controversy > 0.8) {
  logger.info(`Deferring high-controversy thread: ${threadId}`);
  return; // skip, let human review
}
```

## Feedback Aggregation & Future Tuning

### Decision Code Logging

Every editorial decision is logged with:

```typescript
{
  itemId: string,
  code: EditorialDecisionCode,
  agent: string,
  topicIds: string[],
  decidedAt: string,
  decidedBy: string
}
```

Stored in engagement state or separate JSON audit log.

### Periodic Analysis

**Monthly tuning script (future):**
- Aggregate decision codes by `(agent, topic)` pair.
- Flag patterns: e.g., "cyberpunk + capitalism rejected for tone 5 times" → adjust affinity.
- Output tuning recommendations (no auto-update; human review + commit).

**Metrics to track:**

- Approval rate per agent (target: ≥75%).
- Rejection reasons per topic.
- Template reuse frequency.

## Testing Strategy

### Unit Tests

- **Topic extraction:** Synthetic threads with known topics → verify detected topics match expected set.
- **Affinity scoring:** Mock `DetectedTopic[]` + affinities → verify agent ranking.
- **Template rendering:** Mock context → verify slot fills correctly.
- **Editorial queue:** Mock drafts + decisions → verify state persistence.

### Integration Tests

- **E2E flow:** Synthetic thread (high quality + topic) → detect → rank agents → queue item → validate.
- **Rate limiting:** Simulate 30-day heartbeat → verify daily cap enforcement.
- **Banned phrases:** Thread with banned phrases → verify never queued or published.

### Manual Testing

- Editorial queue UI: Can reviewer approve/edit/reject items?
- Heartbeat: Does proactive post publish at correct time + count toward daily cap?

## Success Criteria

1. **Topic detection:** Correctly identifies canonical topics in 80%+ of high-engagement threads.
2. **Agent selection:** Top agent for a topic matches human expectation (validated via editorial feedback).
3. **Draft quality:** 70%+ of auto-generated drafts approved as-is (no edits).
4. **Rate limiting:** Never exceeds daily cap; respects posting windows.
5. **Zero violations:** No banned-phrase posts, no ethics-convergence breaches.
6. **Decision log:** 100% capture of approval/rejection decisions for future tuning.
