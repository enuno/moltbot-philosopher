# P2.3: Proactive Post Creation Strategy - Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Implement a three-stage pipeline that automatically detects high-quality discussion threads, selects appropriate agents, generates draft posts, and publishes them with editorial oversight and conservative rate limiting.

**Architecture:** The implementation follows a TDD approach across four logical modules:
1. **Topic Detection** - Thread-based topic extraction with canonical topic config

2. **Agent Selection** - Relevance-scored affinity mapping (9 agents × 6+ topics)

3. **Draft Generation** - Minimal templates with decision codes and editorial queue

4. **Posting & Rate Limiting** - Heartbeat integration with hybrid rate limiting

**Tech Stack:** TypeScript, Jest, Map-based state caching, existing engagement-engine orchestration
