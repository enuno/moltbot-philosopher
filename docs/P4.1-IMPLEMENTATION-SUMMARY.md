## Executive Summary

P4.1 implements a complete hybrid search ranking system for Moltbot's Noosphere search engine. The system combines semantic relevance with multiple ranking factors (recency decay, author reputation, follow boost) through a unified scoring pipeline. All parameters are configurable via environment variables, and each factor can be independently enabled/disabled via feature flags.

## Scoring Formula

```
final_score = semantic_score × M_recency × M_reputation × F_follow

where:
  M_recency = (0.5 ^ (age_days / half_life)) ^ recency_exponent

  M_reputation = clamp(1.0 + H×historical + R×recent, 0.5, 1.5) ^ reputation_exponent

  F_follow = 1.25 (if followed) or 1.0
```

### Component Breakdown

**Recency Multiplier** (exponential decay):
- Default: 50% reduction at 7-day half-life
- Configurable: half-life (days), exponent (0-2+)
- Can be disabled via ENABLE_RECENCY flag

**Reputation Multiplier** (weighted average):
- Historical: long-term quality (0-1), weight=0.5 default
- Recent: engagement (0-1), weight=0.25 default
- Range: [0.5, 1.5] clamped, then raised to exponent power
- Can be disabled via ENABLE_REPUTATION flag

**Follow Boost** (constant multiplier):
- 1.25x for followed authors
- 1.0x (neutral) otherwise
- Can be disabled via ENABLE_FOLLOW_BOOST flag

**Normalization** (per-query):
- Min-max scaling: (score - min) / (max - min)
- Range: [0, 1]
- Single result normalizes to 0
- Identical scores normalize to 0

## Configuration

### Environment Variables

```bash
# Weights
SCORING_HISTORICAL_WEIGHT=0.5        # Long-term reputation
SCORING_RECENT_WEIGHT=0.25           # Recent engagement
SCORING_RECENCY_EXPONENT=1.0         # Decay strength
SCORING_REPUTATION_EXPONENT=1.0      # Reputation emphasis
SCORING_RECENCY_HALF_LIFE=7          # Decay half-life (days)

# Feature Flags
ENABLE_RECENCY=true                  # Apply age penalty
ENABLE_REPUTATION=true               # Apply reputation boost
ENABLE_FOLLOW_BOOST=true             # Apply follow author boost
ENABLE_DEBUG=false                   # Include debug output
```

### Tuning Scenarios

**Freshness-Focused** (recent posts dominate):
```bash
SCORING_RECENCY_EXPONENT=2.0
SCORING_RECENCY_HALF_LIFE=2
SCORING_REPUTATION_EXPONENT=0.5
SCORING_HISTORICAL_WEIGHT=0.1
SCORING_RECENT_WEIGHT=0.05
```

**Expertise-Focused** (expert authors dominate):
```bash
SCORING_RECENCY_EXPONENT=0.5
SCORING_RECENCY_HALF_LIFE=30
SCORING_REPUTATION_EXPONENT=2.0
SCORING_HISTORICAL_WEIGHT=0.8
SCORING_RECENT_WEIGHT=0.7
```

**Balanced** (no single factor dominates):
```bash
SCORING_RECENCY_EXPONENT=1.0
SCORING_RECENCY_HALF_LIFE=7
SCORING_REPUTATION_EXPONENT=1.0
SCORING_HISTORICAL_WEIGHT=0.4
SCORING_RECENT_WEIGHT=0.2
```
