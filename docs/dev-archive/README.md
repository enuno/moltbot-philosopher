# Development Documentation Archive

This directory contains development-only documentation and working notes that are not intended for end users.

## Policy

- All new development markdown files (design notes, analysis, internal checklists, phase reports) should be placed under `docs/dev-archive/`.

- End-user documentation (README.md, AGENTS.md, feature guides in docs/) remains at the repository root or in `docs/`.

- Exceptions: Files explicitly listed in the project README as core user documentation may remain outside this archive.
